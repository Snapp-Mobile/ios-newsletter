# Articles

## Programming

### [8 Things I Wish I Knew When I Started Programming](https://www.hackingwithswift.com/articles/252/8-things-i-wish-i-knew-when-i-started-programming)

[Paul Hudson](https://bsky.app/profile/twostraws.bsky.social){: .author}

Paul Hudson reflects on 25 years of programming and shares eight lessons he wishes he had learned earlier, from embracing constant change to building resilience through hard bugs. As we know, he emphasizes that deep fundamentals like data structures, debugging habits, and collaboration matter far more than memorizing syntax. He also highlights how we all like to chase new tools, but in our apps and careers, long-term growth comes from learning how to learn and applying knowledge repeatedly. It’s a practical, encouraging read for developers at any stage who want a healthier mindset around progress.

## Swift

### [Working with the Keychain in iOS](https://tanaschita.com/ios-keychain-secure-data-storage/)

[Natascha Fadeeva](https://mastodon.social/@tanaschita){: .author}

The author walks through secure data storage with Apple’s Keychain Services API, focusing on real operations we use in our apps: add, read, update, and delete. The article clearly explains key query fields such as `kSecClass`, `kSecAttrService`, `kSecAttrAccount`, and `kSecValueData`, and why each one matters for predictable behavior. As we know, security APIs can feel low-level, and she acknowledges the C-style ergonomics while showing how to build clean Swift wrappers on top. It’s a solid, practical refresher for handling credentials and tokens safely on iOS.

### [The 2FA app that tells you when you get `314159`](https://blog.jacobstechtavern.com/p/building-a-2fa-app-that-detects-patterns)

[Jacob Bartlett](https://bsky.app/profile/jacobstechtavern.com){: .author}

This article shares the full journey of building a playful 2FA app that detects “interesting” one-time codes and notifies users when rare patterns appear. He combines TOTP generation, Keychain-backed account storage, SwiftUI product work, and on-device notification scheduling into a surprisingly deep engineering case study. As we all do in side projects, he iterates from proof of concept to release, then profiles performance bottlenecks with Instruments and improves throughput using algorithm changes and concurrency. The post is both entertaining and technically rich, especially for anyone interested in iOS optimization and product-minded experimentation.

## UI / UX

### [Refreshing and animating views using TimelineView in SwiftUI](https://nilcoalescing.com/blog/TimelineViewInSwiftUI/)

[Natalia Panferova](https://bsky.app/profile/natpanferova.bsky.social){: .author}

Natalia Panferova explains how `TimelineView` enables time-driven SwiftUI updates without relying on explicit state mutations. She demonstrates practical schedules like `.everyMinute`, `.periodic`, and `.animation`, plus how `TimelineView.Context` gives us `date` and `cadence` to adapt rendering detail. In our apps, this is especially useful for clocks, animated backgrounds, and continuously updating visuals where regular refresh behavior matters. As we know, choosing the right primitive is key, and the article clarifies when `TimelineView` is a better fit than `Timer`.

### [Taming Row Height and Spacing Jumps in SwiftUI List with a Custom Layout](https://fatbobman.com/en/posts/taming-row-height-and-spacing-jumps/)

[Xu Yang](https://bsky.app/profile/fatbobman.com){: .author}

This entry tackles a familiar SwiftUI pain point: jumpy row-height and spacing transitions inside `List` when dynamic content appears or disappears. It proposes a native SwiftUI architecture using `Animatable`, custom `Layout`, `LayoutValueKey`, and state decoupling to animate height smoothly without visual squeezing artifacts. It also introduces a custom vertical stack strategy to keep spacing synchronized with collapse progress, solving the subtle “last-frame jump” many of us see in production UIs. For teams polishing interaction quality in our apps, this is an advanced and very practical animation deep dive.

## AI / ML

### [🍎📚Cupertino](https://github.com/mihaelamj/cupertino)

[Mihaela Mihaljevic](https://github.com/mihaelamj){: .author}

This repository presents Cupertino, a Swift-based local Apple documentation crawler and MCP server designed for AI-assisted development workflows. It combines large-scale documentation ingestion, SQLite FTS5 search, and tool/resource interfaces so agents can query Apple platform knowledge with deterministic, offline-friendly results. As we know, integrating AI into daily engineering needs trustworthy context, and this project focuses on exactly that with clear installation paths for Claude, Codex, Cursor, and more. It’s a compelling utility for developers who want tighter feedback loops and better grounding when building iOS, macOS, and Swift features.

## Testing

### [How to Make Custom Test Assertions in XCTest](https://qualitycoding.org/swift-test-assertions/)

[Jon Reid](https://qualitycoding.org/about/){: .author}

Jon Reid shows how to build custom XCTest assertions in Swift that preserve accurate failure locations using `#filePath` and `#line`. He starts from tuple-testing pain points and demonstrates how a focused helper can produce much clearer diagnostics than generic assertions. In our apps, this pattern helps us keep test code expressive while reducing repetitive boilerplate and improving debugging speed. As we all like to keep feedback loops tight, this is a small technique with outsized value for test quality.

## Tools

### [MiniSwift — Swift Compiler for the Web](https://miniswift.run)

[Uğur Toprakdeviren](https://github.com/toprakdeviren){: .author}

MiniSwift showcases an ambitious in-browser Swift toolchain that compiles Swift to WebAssembly and renders a custom SwiftUI runtime on canvas. The project highlights a full pipeline—from lexer and parser to semantic analysis and IR—plus support for stateful UI behavior, diffing, and even Metal-shader-related workflows in the browser. As we know, experimentation around developer tooling is accelerating, and this demonstrates how far lightweight, agent-friendly environments can go without heavyweight dependencies. It’s an inspiring look at compiler engineering, educational tooling, and the future of interactive Swift exploration.
